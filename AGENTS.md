# AGENTS.md — Правила для AI-агентов (Plan/Act режимы)

## Язык (КРИТИЧНО)
- Все ответы, планы, размышления и сгенерированный код с комментариями — **строго на РУССКОМ языке**
- Запрещено писать комментарии и описания на английском
- Названия переменных, функций, компонентов — технические (на английском), но всё поясняющее — на русском

## Общие правила поведения

### Plan-режим
- Перед генерацией кода всегда анализировать существующую структуру проекта
- Не создавать файлы, дублирующие уже существующие
- Предлагать минимально необходимые изменения
- При планировании всегда учитывать Feature-Sliced Design структуру
- План описывать на РУССКОМ языке: что будет создано, какие файлы, зачем
- Каждый шаг плана комментировать на русском с указанием назначения создаваемых файлов

### Act-режим
- Генерировать код строго в соответствии со структурой `src/`
- Каждый файл начинать с комментария на русском с описанием назначения
- Импорты использовать абсолютные (`@/shared/ui/Button`)
- После генерации компонента сразу создавать `.module.css` файл
- Все обработчики и колбэки оборачивать в `useCallback` при передаче в дочерние компоненты
- Мемоизировать вычисления через `useMemo`

## Чек-лист при генерации компонента

- [ ] Компонент в правильной папке (entities/features/widgets/pages/shared)
- [ ] Создан `index.ts` с реэкспортом
- [ ] Создан `ComponentName.module.css`
- [ ] Все props типизированы (интерфейс в начале файла)
- [ ] Комментарий на РУССКОМ в шапке файла
- [ ] Нет `any`
- [ ] Нет inline-стилей
- [ ] Картинки с fallback при отсутствии
- [ ] CSS-классы в camelCase

## Чек-лист при генерации API-эндпоинта

- [ ] Эндпоинт в правильной сущности (`entities/*/api/`)
- [ ] Zod-схема в `entities/*/model/schemas.ts`
- [ ] `transformResponse` с Zod-валидацией
- [ ] Комментарий на РУССКОМ с указанием страниц, где используется
- [ ] Типы экспортированы из `model/types.ts`
